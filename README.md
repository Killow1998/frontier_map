# 边陲驻泊 (Frontier Map)

本项目包含了魔兽争霸 III 自定义防守地图《边陲驻泊》的 LNI 开源结构数据。我们通过纯文本化的方式管理地图的物体数据与 JASS 脚本，从而实现更优雅、更安全的现代游戏开发工作流（支持 Git 版本控制与多人协作）。

## 🚀 项目实现与核心机制

当前仓库的主要实现形式为：**数据驱动与底层 JASS 覆写**。
为了突破传统 GUI 触发器的限制与性能瓶颈，本项目将许多核心机制转移到了底层代码中：
1. **装备升级系统（概率坍缩）**：通过全局哈希表（YDHT）绑定单位句柄，实现了完全独立的“升级失败补偿”概率积累系统，替代了原有的零散触发器。
2. **马甲伤害自动溯源**：在全局伤害引擎中截获马甲（Dummy）单位，自动追溯真正的雇佣兵主人，从而使法系职业的 AOE 技能能够完美触发吸血、吸蓝等装备特效。
3. **动态数值与 UI 增强**：去除了刷屏式的文字提示，采用动态角度和速度的悬浮飘字；并实现了属性（如智力）与召唤物（蛇棒、地狱火炮台）战斗力的实时、动态挂钩。

## 🛠️ 如何使用本项目

### 环境依赖
1. **魔兽争霸 III 客户端** (推荐 1.24 - 1.27 或以上版本)
2. **W3x2Lni 工具** (用于在 `.w3x` 压缩包和 `.lni` 散文件夹之间进行相互转换)
3. **YDWE 引擎** (如果你想修改地形或查看原始 GUI 触发器)

### W3x2Lni 核心使用教程
由于 `w3x2lni` 具有特定的路径结构与参数规范，以下是开发过程中最常用的两条命令，请在包含 `w3x2lni.exe` 的根目录下执行。

**1. 解包：将 `.w3x` 地图转换为 LNI 结构（用于代码开发与 Git 管理）**
```powershell
.\w3x2lni.exe lni "你的地图文件.w3x" "输出的文件夹名称"
# 例如：
.\w3x2lni.exe lni "边陲驻泊1.9.16.w3x" "base_src"
```

**2. 打包：将 LNI 文件夹编译回 `.w3x` 地图（用于进游戏测试或发布）**
```powershell
.\w3x2lni.exe obj "LNI文件夹名称" "输出的地图名.w3x"
# 例如编译当前版本：
.\w3x2lni.exe obj base_src "边陲驻泊_v1.10.1.w3x"
```

*注意：`w3x2lni` 在提取或打包时，有时会受限于文件路径中的中文字符或执行上下文，如果遇到找不到文件的错误，请尝试使用绝对路径或 `Rename-Item` 临时将中文地图名改为英文（如 `map.w3x`）。*

## 📁 目录结构说明

- `base_src/`：LNI 格式的地图源码核心目录。
  - `map/`：包含地形文件 (`.w3e`), 摆放物 (`.doo`) 和核心代码 (`war3map.j`)。
  - `table/`：对象编辑器数据转化为的文本文件 (`unit.ini`, `item.ini`, `ability.ini` 等)，修改装备数值和描述主要在这里进行。
  - `resource/`：导入的模型、图标与贴图文件。
- `CHANGELOG.md`：详细的版本更新日志。
- `边陲驻泊_vX.X.X.w3x`：通过当前源码直接打包生成的可玩地图文件（作为版本节点保留）。
