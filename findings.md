# Jass to TS 迁移与地图更新代码审查报告

## 1. 核心架构与全局同步
- **全局变量映射**: 采用 `jass.globals` 与 `_G` 双向映射机制，确保 TS 与遗留 JASS 环境状态实时同步。
- **触发器接管**: 实现了完善的 `disableLegacyTrigger` 与 `replaceGlobalTrigger` 机制，通过 `takeover.ts` 的白名单模式精准控制迁移范围。
- **伤害总线**: 为 1.27a 等老版本复刻了“任意单位受伤”分发器，支持事件内伤害写回（`EXSetEventDamage`），避免了生命值近似修改带来的不稳定性。

## 2. 战斗系统与伤害归属
- **马甲伤害脚本化**: 通过 `startProjectileDamageSimulation` 等函数，将原本由马甲单位（Dummy）造成的伤害重定向至施法者本体结算，解决了击杀统计与经验归属问题。
- **伤害类型统一**: 成功将杂乱的伤害类型收口为“物理攻击”、“非攻击物理”、“魔法伤害”三类，符合现代地图开发规范。
- **会心判定链**: `markRecentAttackInfo` 机制使得脚本技能也能正确触发会心、闪避等普攻判定逻辑。

## 3. 流程控制与系统稳定性
- **昼夜系统回正**: 采用“固定跳变模式”，通过真实计时器控制昼夜时长（受难度影响），规避了游戏时间流速修改导致的逻辑错位。
- **Boss AI 优化**: 从每秒轮询切换回“事件驱动”模式（进范围/被攻击触发），显著降低了运行时 CPU 压力，同时保留了概率施法多样性。
- **联机安全性**: 
  - 核心逻辑（复活、任务提交、昼夜切换）移除了不当的 `GetLocalPlayer` 分支。
  - 移除了非必要的 UI 模块（如 `UnitBlood`），降低了因 UI 回调异步导致的分车风险。
  - 增加了初始化幂等锁与进攻波次执行锁，防止因触发器重复调用导致的双刷。

## 4. 物品与合成系统
- **装备智力联动**: 法杖全链新增智力缩放逻辑（魔能汲取、雷杖眩晕、地狱之杖指数增强），提升了法系职业后期成长性。
- **描述美化**: 物品描述通过 `item.ini` 进行了系统性重写，采用分层多色风格，极大提升了装备分支与词条的可读性。
- **测试沙盒**: 完善了传送锚点旁的测试资源投放逻辑，通过显式开关 `udg_enable_migration_test_content` 控制，兼顾了开发测试便利性与正式局安全性。

## 5. 建议与风险点
- **YDWE 依赖**: 部分功能（如面板绿字加成）仍依赖于 YDWE 的二进制能力回退。在完全脱离 YDWE 的环境下需关注 `isAttackBonusAbilityFallbackAvailable` 的表现。
- **内存回收**: 部分周期性计时器（如 `registerInitGuardTimer`）已实现死亡自动销毁，但仍需持续关注大规模战斗下 `texttag` 与 `effect` 的句柄清理频率。
- **联机验证**: 虽然代码层面已规避常见分车点，但由于魔兽引擎特性，仍建议在 4 人满编环境下进行至少一轮完整的通关回归测试。
