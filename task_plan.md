## 已做决策
| 决策 | 理由 |
|------|------|
| 采用 萤火披风 格式 | 用户明确要求对齐该格式（风味说明 -> 属性 -> 特效） |
| 逐一手写内容，脚本批量注入 | 兼顾“禁止均值回归”的高质量文字要求与文件修改的准确性 |
| 为消耗品添加独特设定 | 增加游戏的代入感，例如将“果汁”设定为营地特调补给 |
| 使用 YDHT 存储额外概率 | YDHT 是地图内置的全局哈希表，方便跨触发器存取单位自定义属性且支持异步安全 |
| 定义 EquipmentUpgradeLogic 助手函数 | 减少 JASS 代码冗余，确保所有装备升级逻辑的一致性 |
